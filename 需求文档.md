# 《城市脉搏：公共交通效率分析与优化建议生成器》需求文档

## 一、项目信息

| 项目 | 内容 |
|------|------|
| 项目名称 | 城市脉搏：公共交通效率分析与优化建议生成器 |
| 竞赛组别 | 初中组 |
| 主题 | 数据分析与社会洞察 |
| 适用语言 | Python (Pandas, Matplotlib/Seaborn, Tkinter) |
| 核心考察 | 数据清洗与处理、统计图表可视化、基础算法、报告自动生成 |

---

## 二、项目背景

城市交通拥堵是常见的社会问题。本项目以杭州地铁1号线（简化版14站）为分析对象，模拟并分析地铁线路的运行数据，找出低效环节，自动生成可视化分析报告和优化建议，帮助理解公共交通运营效率问题。

| 4 | 翁梅 | 普通站 | 1.0 |
| 5 | 客运中心 | 换乘站 | 1.3 |

| 13 | 城站 | 火车站 | 1.5 |

### 5.2 数据生成规则

| 规则 | 描述 |
|------|------|
| 时间范围 | 7天（5工作日 + 2周末），每天 06:00-23:00（18个时段） |
| 客流规律 | 工作日双峰高斯（早8:00 σ=0.8，晚18:00 σ=1.0），周末平缓双峰 |
| 延误规律 | 高峰 + 大站 → 延误概率高，基底0.5分钟 |
| 车速规律 | 基准40km/h，拥挤时降至28-32km/h |
| 拥挤度 | 客流/容量（大站5500，普通站4000），上限1.0 |
| 准点标准 | 延误 < 2分钟视为准点 |

## 六、界面原型

### 6.1 整体布局

```
┌─────────────────────────────────────────────────────────┐
│              标题栏 + 导出按钮 (60px)                      │
├──────────┬─────────────────────────────┬────────────────┤
│          │                             │                │
│  控制面板  │       主图表区               │   分析报告区    │
│  220px   │      自适应宽度              │    280px       │
│          │                             │                │
│ · 日期筛选 │   matplotlib 嵌入图表         │  · 数据概览     │
│ · 图表选择 │   640×420 px               │  · 异常检测     │
│ · 站点筛选 │                             │  · 优化建议     │
│ · 操作按钮 │   时间轴滑块 (6-23)          │  · 预测结果     │
│          │                             │                │
│          │   底部缩略图 ×3              │                │
├──────────┴─────────────────────────────┴────────────────┤
│              状态栏 (25px)                                │
└─────────────────────────────────────────────────────────┘
```

### 6.2 交互流程

```
用户启动程序
  │
  ├──→ 自动加载模拟数据 → 显示默认图表（热力图）+ 分析报告
  │
  ├──→ 切换图表类型 → 主图表区刷新
  ├──→ 修改筛选条件 → 所有图表和报告动态更新
  ├──→ 拖动时间滑块 → 图表高亮对应时段
  ├──→ 点击缩略图 → 切换主图表
  │
  ├──→ 点击「导出图表」→ 保存当前图表为 PNG
  ├──→ 点击「导出报告」→ 保存分析报告为 TXT
  └──→ 点击「API获取」→ 调用高德 API 获取真实数据（需配置 key）
```
